# 16. Decisions & Open Questions

> **Purpose of this file.** This is the project's single decision log. Every unresolved question that appears in any spec document is collected here so the team can see the full picture in one place — what is open, who owns it, what it blocks, and what the answer was once it is resolved. When a decision is made, update this file first, then update the referenced doc(s) and any affected code.
>
> **How to use it.** Open questions are grouped by domain. Each entry has an ID (`DQ-NNN`) that other documents can reference instead of repeating the full question. When you resolve a decision, move it to the **Resolved Decisions** section at the bottom of this file, fill in the resolution date and final answer, and list which docs need updating.
>
> **Who maintains this file.** Whoever makes or witnesses a decision adds the resolution here. If you resolve something in a meeting, Slack thread, or PR comment, it must be recorded here within 24 hours — otherwise the rest of the team (and any AI coding agents working from these docs) will continue treating it as unresolved.

## 1. How to Read the Tables

Each open question entry contains:

| Field | Meaning |
|---|---|
| **ID** | Stable reference (`DQ-NNN`) used in other documents. Never reuse a retired ID. |
| **Question** | What needs to be decided, written as a concrete question. |
| **Why it matters / what it blocks** | The exact work that cannot start or cannot be finished until this is resolved. Be specific — "blocks agent code" is less useful than "blocks `backend/app/agents/delivery_performance.py` and all LLM prompt templates." |
| **Options / candidates** | Known choices being considered. Not exhaustive — add more as they emerge. |
| **Owner** | The person (by role) who has the authority and context to make this call. One person only — not a committee. |
| **Priority** | `P1` = blocks Phase 1 code; `P2` = needed before production go-live but not for development to start; `P3` = can defer to Phase 2. |
| **Source doc(s)** | Where the open question appears in the project documentation. |

When resolved, move the entire entry to §11, add `Resolved:`, `Resolution date:`, `Final answer:`, and `Docs to update:`.

### DQ-021 — LLM provider and model

**Question:** Which LLM vendor and which specific model will power the three Phase 1 agents (Delivery Performance, Quality Intelligence, Client Interaction)?

**Owner:** Engineering Lead + BSG Leadership
